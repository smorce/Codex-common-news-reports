## [メモ] Dreaming:より役立つChatGPTのための優れた記憶システム

### Executive Summary
- OpenAI の ChatGPT メモリ改善に関する公式記事を、筆者が要点整理したスクラップである。
- 新しいメモリシステムは、会話間で文脈を引き継ぎ、時間経過に応じて関連性を保つことを狙っている。
- 従来の保存済み記憶は、明示的に記録された断片を扱う傾向があり、鮮度低下や不正確化が課題だった。
- Dreaming は多数の会話をバックグラウンドで参照し、記憶状態を自動的に統合・更新する手法として説明されている。
- 評価観点は、有用な文脈の維持、ユーザーの好みや制約への追従、時間経過への適応の三つに整理されている。
- 旅行や機材選定、食事提案などの例を通じ、記憶がある場合の応答の具体性が示されている。
- 米国の Plus / Pro ユーザーから展開され、Free / Go や他国への展開も予定されていると紹介されている。
- 筆者は、断片検索ではなく大きなコンテキスト全体を整合的に更新する点を Dreaming の重要点と見ている。

## 「LFM2.5-Audio-1.5B-JP」を試す

### Executive Summary
- Liquid AI の日本語対応音声モデル LFM2.5-Audio-1.5B-JP を試した記録である。
- 同モデルは 1.5B 規模の Speech-to-Speech モデルで、ASR や TTS を独立部品として必要としない構成と紹介されている。
- 音声入力評価では VoiceBench や ASR の CER 比較が引用され、軽量モデルとしての性能が整理されている。
- Mac 向け GGUF 版では、独自 CLI とサーバーバイナリが配布されており、llama.cpp 本体とは別の実行経路が使われた。
- GGUF 版の ASR では固定システムプロンプトの制約が見つかり、日本語指定が受け付けられない挙動が確認された。
- 純正 liquid-audio パッケージでは ASR、TTS、インターリーブ音声チャットのサンプルを動かし、短い音声では高速に推論できた。
- 長い音声では出力が壊れたりループする傾向があり、モデルサイズや会話履歴の影響が課題として観察された。
- 筆者は、このサイズで S2S が動く点に可能性を見つつ、用途には制約があるとまとめている。

### Key Findings
- LFM2.5-Audio-1.5B-JP は日本語対応の軽量 Speech-to-Speech モデルである。 [^]
  - Footnote: モデル説明に「Liquid AI初となる日本語対応の音声モデル」「1.5B」「日本語のSpeech-to-Speechモデル」とある。
- 独立した ASR や TTS コンポーネントなしに音声・テキストを扱う設計である。 [^]
  - Footnote: 本文では「独立したASR（自動音声認識）やTTS（音声合成）のコンポーネントを必要としません」と引用されている。
- 生成方式はリアルタイム対話向けのインターリーブド生成と、ASR/TTS 向けのシーケンシャル生成に分かれる。 [^]
  - Footnote: 「インターリーブド（交互）生成」と「シーケンシャル（逐次）生成」の二方式が説明されている。
- GGUF 版には専用の CLI とサーバーが同梱されていた。 [^]
  - Footnote: 筆者は展開結果として「llama-liquid-audio-cli」「llama-liquid-audio-server」を確認している。
- GGUF 版の ASR では日本語指定付きシステムプロンプトが受け付けられなかった。 [^]
  - Footnote: 実行結果に「Unsupported system prompt. Supported prompts are」と表示され、`Perform ASR in japanese.` が拒否されている。
- 純正 Python パッケージでは短い日本語音声の ASR が 0.425 秒程度で生成された。 [^]
  - Footnote: ASR 実行結果に「elapsed: 0.425 secs」と記録されている。
- 5分程度の長い音声では出力がループする問題が観察された。 [^]
  - Footnote: 筆者は「5分に伸ばすとこんな感じで出力がループしてしまった」と述べ、反復出力を示している。
- TTS は動作したが、声質変更の自由度には疑問が残った。 [^]
  - Footnote: TTS の後に「声は変更できないのかな？何度か実行してみたけど、声質は常に同じに思える」とある。
- マルチターン音声チャットでは過去会話に引っ張られる挙動があった。 [^]
  - Footnote: 筆者は「過去の会話に引っ張られる」「同じことを繰り返したりする」と観察している。
- 日本語 S2S のファインチューニングでは、テキスト6個に対して音声9個のインターリーブ比率が示されている。 [^]
  - Footnote: モデルカード引用として「interleaved_text_tokens=6」「interleaved_audio_tokens=9」と記載されている。
