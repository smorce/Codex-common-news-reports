## 「TabFM」を試す

### Executive Summary
- Google の表形式データ向け基盤モデル TabFM を試したスクラップである。
- TabFM は分類と回帰を対象に、未知の表データへ単一の順方向パスで予測を返すゼロショット型モデルとして紹介されている。
- 従来の XGBoost などで必要になりがちな前処理、特徴量設計、ハイパーパラメータ調整の負担を減らす方向性が示されている。
- アーキテクチャは行と列の交互アテンション、行圧縮、ICL Transformer を組み合わせる構成として説明されている。
- 学習データは実企業データではなく、構造的因果モデルで生成した数億規模の合成テーブルを使う点が特徴である。
- Colaboratory 上で PyTorch バックエンドを使い、分類、回帰、Titanic、California Housing の例を実行している。
- 便利さは評価しつつ、モデル重みが非商用ライセンスである点と、大きなデータでは計算時間やメモリに注意が必要と整理している。

### Key Findings

  - Footnote: 本文冒頭で「表形式データの分類および回帰のために特別に設計された基盤モデル」と紹介されている。

  - Footnote: モデルカード抜粋に「ファインチューニングやハイパーパラメータ調整は不要」「単一の順方向パスで予測」とある。

  - Footnote: 利用可能なチェックポイントと制約事項で「分類タスク（最大10クラス）」および「最大クラス数は10クラスまで」と記載されている。

## PostgreSQL で N-gram全文検索（pg_bigm / pg_trgm）

### Executive Summary
- Colaboratory 上で PostgreSQL 16 を起動し、pg_bigm と pg_trgm による N-gram 全文検索を比較したスクラップである。
- 初期状態では pg_trgm のみ利用可能で、pg_bigm は GitHub から取得してビルド、インストールしている。
- pg_bigm は shared_preload_libraries に追加する必要があり、ALTER SYSTEM 後に PostgreSQL を再起動して有効化している。
- 青空文庫の作家別作品一覧 CSV を使い、作品名、読み、作者名などから検索用テキストを作成している。
- 同じ 19474 件のデータをバイグラム用テーブルとトリグラム用テーブルへ投入し、それぞれ GIN インデックスを作成している。
- 2 文字検索では pg_bigm が効率よく 9 件を拾う一方、pg_trgm は全件近くを拾って再チェックする挙動が観察されている。
- 3 文字検索や類似度検索も試し、N-gram 検索の用途にはまだ活用余地があるとまとめている。

### Key Findings
- 検証環境は PostgreSQL 16.14 である。 [^]
  - Footnote: version() の出力に「PostgreSQL 16.14 ... 64-bit」と表示されている。
- 初期の拡張一覧では pg_trgm だけがあり、pg_bigm は追加インストールが必要だった。 [^]
  - Footnote: 最初の pg_available_extensions 確認では「pg_trgm | 1.6」のみが表示され、本文に「pg_bigm がないので、追加する」とある。
- pg_bigm 追加後は pg_bigm 1.2 と pg_trgm 1.6 が利用可能になった。 [^]
  - Footnote: 再確認の出力に「pg_bigm | 1.2」「pg_trgm | 1.6」と表示されている。
- pg_bigm は shared_preload_libraries に設定して PostgreSQL を再起動する必要がある。 [^]
  - Footnote: 本文に「pg_bigm は shared_preload_libraries に追加する必要がある」「サーバの再起動は必要」と記載されている。
