## メモ: Qwen Cloud

### Executive Summary
- Qwen Cloud は Alibaba Cloud とは別枠に見える、Qwen 系モデル API を中心にした AI 向けプラットフォームとして紹介されている。
- 記事では、クラウドの主な利用者が人間の開発者から AI エージェントへ移りつつある、という前提が置かれている。
- 入口は Website、Skills、CLI の3系統で、人間の探索、エージェントの行動、自動化ワークフローを分けて扱う設計になっている。

- 筆者は、実態としてはクラウドインフラ全体ではなく、エージェント向けインターフェースを備えたモデル API プラットフォームに近いと見ている。

## メモ: Gemma 4 Quantization-Aware Training（QAT）
- Date: 2026-06-08T04:34:08+00:00

### Executive Summary
- Google が Gemma 4 の QAT チェックポイントを Hugging Face で公開したことを扱うスクラップである。
- QAT は学習後に圧縮する PTQ と異なり、量子化される前提を学習過程に組み込むため、品質劣化を抑えやすいと説明されている。
- 公開対象には Q4_0、GGUF、モバイル最適化版、Compressed Tensors など複数の形式が含まれる。
- Gemma 4 E2B では、テキスト専用構成で 1GB 以下のメモリ動作が可能とされ、スマホやノート PC での利用可能性が示されている。
- モバイル向けには静的活性化、チャネル単位量子化、部分的 2bit 化、Embedding と KV キャッシュ最適化が挙げられている。
- 対応エコシステムとして llama.cpp、Ollama、LM Studio、vLLM、MLX、LiteRT-LM、Transformers.js などが言及されている。
- 筆者は QAT と通常量子化の体感差には慎重だが、低リソース環境向けの E2B/E4B モバイル最適化版には関心を示している。

### Key Findings
- Gemma 4 の全モデルサイズとドラフターが QAT で最適化されたと紹介されている。 [^]
  - Footnote: 本文冒頭に「すべての Gemma 4 モデルサイズとそのドラフターが、QAT で最適化」とある。
- QAT は圧縮を学習プロセスに直接組み込むため、推論能力を維持しつつサイズ削減を狙う。 [^]
  - Footnote: 本文に「QATは圧縮をトレーニングプロセスに直接組み込み、Gemma 4から期待される推論能力を維持しつつモデルサイズを縮小」とある。
- E2B/E4B 向けにモバイル最適化版が用意され、エッジ環境での利用が重視されている。 [^]
  - Footnote: 本文に「モバイル最適化版（wNa8o8）」「Gemma 4 E2BおよびE4Bモデルで利用可能」と記載されている。
- モバイル最適化では、静的活性化、チャネル単位量子化、2bit 化、KV キャッシュ最適化が重要要素になっている。 [^]
  - Footnote: 本文に「Static activations」「Channel-wise quantization」「Targeted 2-bit quantization」「Embedding / KV キャッシュ最適化」とある。
- Q4_0 や GGUF、Compressed Tensors など、用途別に複数の配布形式がある。 [^]
  - Footnote: 本文に「非量子化QATチェックポイント（Q4_0）」「GGUF形式（Q4_0）」「Compressed Tensors形式（w4a16）」と列挙されている。
- 筆者は、人間が明確に違いを感じるほどの精度差が出るかは不明と見ている。 [^]
  - Footnote: 本文に「精度的にそこまで大きな違いが出るものかどうかはわからない」「人が明確に違いを感じ取れるほどの差は出ないのではないか」とある。
- Unsloth の動的量子化も比較対象として挙げられ、QAT 版だけが唯一の選択肢ではないとされる。 [^]
  - Footnote: 本文に「Unsloth曰く、単純に量子化するよりもUnslothの動的量子化のほうがいい」とある。
